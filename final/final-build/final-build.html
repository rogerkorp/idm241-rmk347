Final Description

Settings

Triggers

  • User hovers over an interactable element.
  • User clicks on a link
  • User clicks on a button.

Rules

  • Whenever the user hovers over an interactable element, thier cursor changes to a pointer.

    • MENU MODE RULES:
  • If the user clicks on the link labeled "settings", then they will enter menu mode
  • Script changes the class and display style of the menu div.
  • Script changes the class and display style of a div located behind the menu.
  • The menu starts moving up from the bottom off screen, and then settles in at the center.
  • The background div fades in, resulting in the content underneath the menu to appear darker.
  • User can click on and enable checkboxes that correspond to script settings
  • To exit out and revert back to normal mode the user can click on either a submit button or a cross symbol.
  • If the user chooses to click the cross, then the settings will not be applied to the calculation
  • If the user chooses to click the submit button, then the settings will be applied.
  • While reverting back to normal mode, both the menu and background divs will be given thier original classes and display styles
  • The menu sinks below the screen, and the background lightens back up

    • SUBMIT BUTTON RULES:
  • If the user clicks on the submit button, the button will play out a growth and shrinking animation
  • On click, the submit button will be given a new class and then – after its animation plays out – it will automatically revert back to it's original class.
  • A breif tone will play to indicate that a calculation was made.
  • After the event occurs, the script will run through a calculation based on the applied settings
  • When the calculation determines the variable that needs to be returned, it will print out the result inside the console log

Loops & Modes

  • Menu Mode allows user to tweak word calculation, but doesn't allow the user to run the calculation until they revert back to normal mode.
